# SEO-аудит сайта ElectrikaVRN

Дата: 2026-02-22

## 1) Есть ли перелинковка?

Короткий ответ: **перелинковка есть, но она несистемная и местами технически неконсистентная**.

Что уже хорошо:
- В статьях есть блоки «Похожие статьи», то есть есть связи между материалами.
- Есть сквозная навигация (header) на ключевые разделы сайта: услуги, калькуляторы, проекты, галерея, статьи, контакты.

Что мешает эффективности:
- Внутри статей одновременно используются разные форматы URL (`.php`, `.html`, «чистые» URL), из-за чего вес может распыляться между дублями URL.
- В хлебных крошках статей встречаются ссылки на `../articles.php`, которого нет в структуре репозитория (признак потенциально битой/устаревшей ссылки).
- В footer встречаются старые `.html`-ссылки на разделы, тогда как в меню используются «чистые» URL.

Вывод: перелинковка формально есть, но **структура URL и единообразие ссылок требуют нормализации**.

## 2) Отдают ли статьи «вес» сайту?

Короткий ответ: **частично да, но не в полной мере**.

Плюсы:
- У статей есть `title`, `description`, canonical, OpenGraph и JSON-LD (`Article`) — это хорошая техническая база.
- Есть ссылки между статьями, что создает внутренний граф ссылок.

Ограничения:
- В видимых блоках перелинковки статьи в основном ссылаются на другие статьи, а не на коммерческие страницы (`/services`, `/calculate`, проектные страницы) с релевантными анкорами. Из-за этого «вес» хуже перетекает в страницы, которые должны ранжироваться по коммерческим запросам.
- Несогласованные URL (`.php/.html/clean`) потенциально делят сигналы между разными адресами.
- В `robots.txt` много нерелевантных WordPress-правил, а также запрет `Disallow: /?`, что потенциально конфликтует с URL-подходом через query-параметры.

Вывод: статьи индексируемы и могут передавать вес, но **сейчас передача ослаблена архитектурой внутренних ссылок и URL-консистентностью**.

## 3) Что не хватает для выхода в топ

### Критично (High impact)
1. **Единая URL-стратегия + 301-редиректы**
   - Выбрать единственный канонический формат (рекомендую «чистые» URL без `.php/.html`) и жестко редиректить остальные версии.
2. **Коммерческая перелинковка из статей**
   - В каждую статью добавить 2–4 контекстные ссылки на коммерческие страницы (услуги, калькуляторы, релевантные проекты) с естественными анкорами.
3. **Локальная SEO-оптимизация (Воронеж + районы/услуги)**
   - Создать/расширить посадочные страницы под кластеры интента: «электрик в Воронеже», «замена проводки в квартире Воронеж», «срочный вызов электрика» и т.д.
4. **Техническая чистка robots/sitemap**
   - Удалить «чужие» WordPress-правила.
   - В sitemap оставить только реально существующие канонические URL.

### Важно (Medium impact)
5. **Topic clusters / контент-хабы**
   - Пиллар-страницы + связанные статьи под единый кластер (безопасность, замена проводки, щиты/автоматика, заземление).
6. **Усиление E-E-A-T**
   - Авторство статей, опыт, кейсы до/после, подтверждение квалификации, расширенные контакты и реквизиты.
7. **CTR-оптимизация сниппетов**
   - Переписать часть title/description под выгоду + локальность + срок/ценность.

### Дополнительно (Low/Medium)
8. Core Web Vitals (LCP/CLS/INP), оптимизация изображений/критического CSS.
9. FAQ-разметка на коммерческих страницах.
10. План внешних упоминаний/ссылок (локальные каталоги, партнерские публикации, региональные площадки).

## 4) Пошаговый план улучшений

### Этап 1 (1–2 недели): Технический фундамент
- Привести все внутренние ссылки к одному формату URL.
- Настроить 301-редиректы между дублями URL.
- Обновить `sitemap.xml` (только канонические живые URL).
- Пересобрать `robots.txt` под реальный стек сайта.
- Прогнать краулер и устранить 404/301-цепочки.

KPI этапа:
- 0 внутренних ссылок на неканонические URL.
- 0 битых ссылок (4xx) в основном графе.
- 100% URL в sitemap отдают 200 и self-canonical.

### Этап 2 (2–4 недели): Перелинковка и коммерческий поток веса
- Для каждой статьи: добавить блок «Рекомендуем услугу» + 2 контекстные ссылки в теле.
- Для каждого сервиса: добавить блок «Полезные статьи по теме».
- Для проектов: добавить взаимные ссылки «проект ↔ услуга ↔ статья».
- Унифицировать анкоры (без переспама).

KPI этапа:
- Не менее 3 ссылок из каждой статьи на коммерческие/транзакционные страницы.
- Рост внутренних переходов из статей в услуги/калькулятор.

### Этап 3 (1–2 месяца): Семантика и контент
- Собрать кластерную семантику по Воронежу (инфо + коммерция).
- Запустить контент-план 8–12 материалов/месяц по кластерам.
- Сделать 6–10 новых регионально-коммерческих посадок.

KPI этапа:
- Рост числа запросов в топ-10/top-20.
- Рост органического трафика на коммерческие страницы.

### Этап 4 (постоянно): Конверсия и доверие
- Добавить сильные CTA внутри статей.
- Усилить соцдоказательства (кейсы, отзывы, фото работ).
- Тестировать формы/кнопки для роста лидов из органики.

KPI этапа:
- Рост CR (органика → заявка/звонок).
- Снижение отказов на ключевых посадках.

## 5) Приоритетный чек-лист (коротко)

1. Нормализовать URL и редиректы.
2. Почистить robots/sitemap.
3. Исправить устаревшие внутренние ссылки (`.html/.php`, несуществующие адреса).
4. Перестроить внутреннюю перелинковку «статья → коммерция».
5. Масштабировать контент-кластеры под локальный спрос.

---

Итог: сайт уже имеет рабочую SEO-базу (мета, schema, контент), но для выхода в топ критично **навести порядок в URL/индексации и направить вес из статей в коммерческие посадки**.
